# 게임 방향 전면 감사 (Game Direction Audit)

> 작성 배경: 21개 페이즈를 지나며 "게임 개발이 산으로 가고 있는 것 같다"는 오너의 문제 제기.
> 이 문서는 기능 단위 리뷰가 아니라 **"이것이 게임인가?"** 를 묻는 큰 틀의 감사이며,
> 모든 판정은 코드에서 직접 확인한 사실(grep/실행 검증)에 근거한다.
> 결론부터: **직감이 맞다. 시스템은 많은데 게임이 없다. 교정 방향을 아래에 제시한다.**
>
> 작성일: 2026-07-10 (master = PR #98 Inner Life 골격 머지 직후, 테스트 435개)

---

## 1부. 진단 — 코드로 확인한 사실

### 🔴 사실 1. 이 게임은 **질 수 없다** (패배 조건 부재)
- `hull`이 0이 되어도 **아무 일도 일어나지 않는다** — 0에서 클램프될 뿐, 어떤 코드도 `hull <= 0`을 검사하지 않는다.
- `oxygen` 0도 동일. 승무원 질식도, 경고 이상의 어떤 결과도 없다.
- 승무원 전원 사망 시 전투 버튼이 비활성화될 뿐 **게임은 태연히 계속된다.**
- 유일한 실패 비슷한 상태(연료 0 = 표류)조차 **무료 "긴급 보급 수신" 버튼 한 번으로 즉시 탈출**된다.

**의미**: 실패가 불가능하면 모든 시스템이 무의미해진다. 위기 대응·자동 수리 정책·치료·보고서 —
이 모든 것은 "잃을 것"이 있을 때만 의미가 있다. 지금은 전부 **결과 없는 숫자 관리**다.
이것이 "재미없음"의 **단일 최대 원인**이다.

### 🔴 사실 2. 이 게임은 **이길 수도 없다** (목표 부재)
- 승리/엔딩/캠페인 목표 관련 코드: **0건.**
- 관문(exit)으로 다음 섹터에 가도 `generateSector`는 `sectorIndex`를 **받지도 않는다** —
  10번째 섹터도 1번째와 동일한 danger 1~5 분포. **진행해도 세계가 강해지지 않고, 도달할 곳도 없다.**

**의미**: "왜 플레이하는가?"에 대한 답이 게임 안에 없다. 세션을 끝내는 자연스러운 순간도,
다음 세션을 시작할 이유도 없다.

### 🟠 사실 3. 고립·미연결 시스템이 쌓여 있다 (폭은 넓고 깊이가 없음)
| 시스템 | 상태 | 근거 |
|---|---|---|
| **스킬트리** | **완전 고립** — 포인트를 찍어도 게임플레이 효과 0 | `skillStore`/`data/skills.js`를 읽는 곳이 UI 3곳(SkillTree/Menu/Overview 표시)뿐, systems 참조 0건 |
| 사냥(Hunting) | 루프 연결 약함 — 보상이 코어 경제에 미미하게만 관여 | Combat 탭에 병합 예정이던 PR #76도 방치 중 |
| 컬렉터/카드 | 카드는 연결됨(cardEffects), 컬렉션 자체는 장식성 | — |
| 무드/관계 (PR #98) | 작동하나 **비가시** — 별도 검토 완료 | `PHASE_21_REVIEW_REPORT.md` D1~D5 |
| 팩션 평판 | 임무 게이트로 일부 연결, 얕음 | — |

**의미**: 페이즈마다 "시스템 하나 추가"가 반복된 결과, 서로 대화하지 않는 부품이 쌓였다.
플레이어 입장에서는 "누르면 뭔가 되긴 하는데 왜 하는지 모르는 버튼"이 늘어난 것.

### 🟠 사실 4. "재미 검증" 프로세스가 없었다
- 지금까지의 모든 검증은 **"돌아가는가"**(테스트 435개, Playwright 스모크)였다. 전부 필요한 것이었지만,
  **"30분 플레이하면 재밌는가"를 물은 적은 단 한 번도 없다.** 플레이테스트 0회.
- 온보딩 0 — 신규 플레이어는 밀도 높은 FM식 대시보드에 설명 없이 던져진다.

### ✅ 잘된 것 (버릴 필요 없는 자산)
- 아키텍처·테스트·세이브 호환·문서화는 건강하다. **엔진은 훌륭하다. 게임이 없을 뿐이다.**
- 코어 루프의 부품 자체(항해→조우→전투→보상→정비→승무원)는 전부 존재하고 작동한다.
- 정책·보고서 시스템은 "패배가 존재하는 게임"이 되는 순간 즉시 가치가 살아난다.

## 2부. 원인 — 어쩌다 산으로 갔나

1. **로드맵이 "시스템 목록"이었다.** Phase N = 시스템 N 추가. "게임을 게임으로 만드는 것"(승패·목표·압박)은
   어느 페이즈에도 없었고, 온보딩/승리 조건은 줄곧 "소형 백로그"에 처박혀 있었다. **이건 백로그가 아니라 본체다.**
2. **완성 판정이 "머지됨"이었다.** 테스트 통과 = 완료로 취급했고, 플레이 경험 검증이 정의에 없었다.
3. **더 쉬운 일(시스템 추가)이 더 중요한 일(게임화)을 계속 밀어냈다.**

## 3부. 교정 계획 — Phase 22 "Make It A Game"

### 대원칙: **신규 시스템 동결 (Feature Freeze)**
아래 22가 끝날 때까지 새 시뮬레이션 시스템을 추가하지 않는다.
Phase 21 잔여(21-D′~F′ 무드 가시화/드라마)도 **뒤로 미룬다** — 죽지 않는 게임에 감정 시스템부터
붙이는 것은 순서가 틀렸다. (PR #98 골격은 이미 머지됨 — 그대로 두되 후속은 보류.)

### 22-A. 패배를 만든다 (최우선)
- **hull 0 = 함선 파괴.** 게임오버 화면(원인 요약 + 항해 기록 통계) → 새 게임. 세이브 초기화.
- **oxygen 0 = 승무원 순차 질식** (일정 시간 지속 시 사망 진행) → 전원 사망 = 게임오버.
- **전원 사망 = 게임오버.**
- **표류 긴급 보급 유료화/제한**(예: 크레딧 비용 + 횟수 제한 or 평판 차감) — 공짜 탈출 버튼 제거.
- 게임오버 직전 상태는 위기·경고·보고서가 이미 알려주고 있음(기존 시스템이 드디어 의미를 갖는 지점).
- 구현 위치: gameClock에 종료 조건 검사 1지점 + gameStore에 `gameOver` 상태 + 오버레이 1개. 침습 최소.

### 22-B. 목표와 진행을 만든다
- **캠페인 목표**: "N번째 섹터의 최종 관문 도달"(1차 엔딩으로 충분 — 예: 섹터 5). 도달 시 승리 화면 + 기록.
- **섹터 난이도 스케일링**: `generateSector(seed, sectorIndex)` — 섹터가 깊어질수록 danger 분포·조우 위험·
  보상이 함께 상승. 적 함대 risk 상한도 sectorIndex와 연동(현재 risk 7 함대는 등장 자체가 불가능한 죽은 콘텐츠).
- **홈에 "현재 목표" 카드**: 다음 목표(관문까지 N노드, 섹터 M/5)를 항상 표시.

### 22-C. 고립 시스템 연결 또는 제거
- **스킬트리 → 실효과 연결**(1순위 — UI·포인트 경제가 이미 있으니 연결만): 각 스킬이 이미 설명하는
  보너스(전투력/회피/보상 등)를 실제 공식에 반영. `data/skills.js`의 기존 서술 = 계약으로 사용.
- **사냥**: 보상을 코어 경제(폐자재/식량/크레딧)에 명확히 편입하거나, PR #76(전투 탭 병합) 방향 재검토.
- 판단 기준: **"이 시스템을 지우면 플레이어가 아쉬워하는가?"** — 아니라면 연결하거나 줄인다.

### 22-D. 첫 10분을 만든다
- 미니 온보딩: 첫 실행 시 3~5단계 가이드(목표 확인 → 항로 결재 → 조우 대응 → 보고서 확인).
- 게임오버/승리 화면이 곧 재시작 온보딩이 되도록 설계.

### 22-E. 재미 검증 루프를 만든다 (프로세스 교정)
- **플레이테스트를 완성 정의에 추가**: 이후 모든 게임플레이 PR은 "30분 자동 플레이 세션 로그"
  (Playwright 자동 플레이 스크립트: 목표 추구 행동 + 세션 통계 — 파산/사망/지루 구간 리포트) +
  가능하면 오너 직접 플레이 피드백을 요구.
- 22-A~D 완료 시점에 **오너 플레이테스트 1회**를 공식 마일스톤으로: "이제 지는 게임인가? 목표가 느껴지는가?"

### 실행 순서와 이후
```
22-A 패배 (1~2 PR) → 22-B 목표·난이도 (1~2 PR) → 22-C 스킬트리 연결 (1 PR) → 22-D 온보딩 (1 PR)
→ 22-E 오너 플레이테스트 → 피드백 반영 라운드
→ (그 후에야) Phase 21 잔여 무드/관계 재개 → 사냥/컬렉터 정리 → 함대
```
- 프로세스는 기존 그대로(작은 PR·테스트·CI·감독 검수). 단 **게임플레이 PR의 완성 정의에
  "플레이 검증" 추가**(§22-E).
- PROJECT_GUIDE §7 로드맵은 이 문서 기준으로 갱신됨.

### 성공 판정 (Phase 22가 끝났다의 정의)
1. 부주의하게 플레이하면 **실제로 게임오버가 난다.** (그리고 그 과정이 위기·보고서로 예고된다)
2. 홈 화면만 보고 "지금 뭘 향해 가는지" 말할 수 있다.
3. 섹터가 깊어질수록 체감 난이도·보상이 오른다. risk 6~7 함대를 실제로 만난다.
4. 스킬 포인트를 찍으면 눈에 보이는 차이가 생긴다.
5. 오너가 30분 플레이 후 "한 판 더"라고 말한다. ← 최종 기준
